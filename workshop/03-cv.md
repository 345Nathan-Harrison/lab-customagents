# Hands-on Lab: Build a CV Agent (Skills)

## Overview

In this lab you will create a **GitHub Copilot Custom Agent** called the **CV Agent**. This agent uses the **skills pattern**: a single agent is enhanced with reusable skill files that give it specialised capabilities. The CV Agent takes rough career notes and produces a polished, ATS-friendly CV in multiple formats using a **formatting skill** and a **tailoring skill**.

### What you will learn

- How to build an agent that uses **reusable skill files** (`.md` files in a skills folder).
- How to write a **requirements document** for a skills-based agent.
- How to create an **`.agent.md`** file that references skill files.
- How to test the finished agent inside VS Code.

## Step 1 — Clone the Starter Repository

1. Open a **terminal** and navigate to your projects folder:

```powershell
cd c:\dev
```

2. Clone the lab repository:

```powershell
git clone https://github.com/345Nathan-Harrison/lab-customagents-init.git
cd lab-customagents-init
```

3. Remove the .Git folder:

```powershell
Remove-Item -Recurse -Force .git
```

## Step 2 — Open the Repository in VS Code

1. Run `code .`
2. Confirm `.github/agents/` and `.github/instructions/custom-agent.instructions.md` exist.
3. Open **Copilot Chat** and confirm it is active.

## Step 3 — Read the Instruction File (Optional but Recommended)

Skim `.github/instructions/custom-agent.instructions.md`. Pay special attention to the **Skills pattern** section — agents gain capabilities through skill files.

## Step 4 — Use Copilot to Write the Requirements Document

### 4.1 — Ask Copilot to draft the requirements

1. Open **Copilot Chat** in **Plan** mode.
2. Type:

   ```
   I want to build a GitHub Copilot custom agent called "CV Agent" that
   uses the skills pattern.

   The agent:
   - Receives rough career notes from the user: work history, education,
     skills, achievements, certifications
   - Produces a polished CV/resume in Markdown format

   It uses 2 skill files:
   1. **CV Formatting Skill** (.github/instructions/cv-formatting.instructions.md)
      - Knows how to structure a CV: header, professional summary,
        experience (reverse chronological), education, skills, certifications
      - Enforces formatting rules: action verbs, quantified achievements,
        consistent date format, concise bullet points
      - Ensures ATS-friendliness (no tables, no images, standard headings)
   2. **CV Tailoring Skill** (.github/instructions/cv-tailoring.instructions.md)
      - When given a job description, adapts the CV to highlight relevant
        experience and skills
      - Reorders and emphasises bullet points that match the job requirements
      - Adds a "Key Skills" section mapped to the job description keywords

   Output saved to ./output/cv.md

   I want a requirements markdown file for this agent.
   Include:
   - Agent description
   - Skill file descriptions
   - Functional and non-functional requirements
   - Example input and output
   ```

3. Save to `cv-requirements.md`.

### 4.2 — Review and refine

```
Add a constraint that the CV must be under 2 pages (roughly 800 words).
The agent should prioritise recent experience (last 5-7 years) and
summarise older roles in 1-2 lines. Also add a professional summary
generator that creates a 3-sentence overview based on career trajectory.
```

## Step 5 — Use Copilot to Create the Agent and Skill Files

### 5.1 — Create the CV Formatting skill

1. Create `.github/instructions/cv-formatting.instructions.md`
2. Ask Copilot:

   ```
   Using #file:cv-requirements.md generate the "CV Formatting Skill"
   instruction file. This should contain formatting rules, section
   ordering, and ATS-friendliness guidelines. Add applyTo frontmatter
   targeting the CV agent.
   ```

### 5.2 — Create the CV Tailoring skill

1. Create `.github/instructions/cv-tailoring.instructions.md`
2. Ask Copilot:

   ```
   Using #file:cv-requirements.md generate the "CV Tailoring Skill"
   instruction file. This should contain rules for adapting a CV to
   match a specific job description. Add applyTo frontmatter targeting
   the CV agent.
   ```

### 5.3 — Create the agent

1. Create `.github/agents/cv.agent.md`
2. Ask Copilot:

   ```
   Using #file:cv-requirements.md generate a "CV Agent" that references
   the cv-formatting and cv-tailoring instruction files.
   Model: Claude Sonnet 4.5.
   Use a 3-phase workflow: Extract → Format → Polish.
   ```

### 5.4 — Review the agent and skills

| Element          | What to check                                              |
| ---------------- | ---------------------------------------------------------- |
| **Agent**        | References both skill files via `#file:` or instruction paths. |
| **Formatting**   | ATS rules, section order, action verbs, quantified achievements. |
| **Tailoring**    | Keyword matching, relevance scoring, reordering logic.     |
| **Tools**        | Must be `["read", "edit"]`.                                |

## Step 6 — Test the CV Agent

### 6.1 — Invoke the agent (general CV)

1. Select **CV** from the agent dropdown.
2. Provide your career notes:

   ```
   Here are my rough career notes:

   - Currently: Senior Software Engineer at Contoso Ltd (2021-present)
     Built microservices in Go, led team of 4, migrated monolith to k8s,
     reduced deploy time from 2 hours to 15 minutes
   - Previously: Software Engineer at Fabrikam Inc (2018-2021)
     Full-stack TypeScript, React frontend, Node.js APIs, PostgreSQL
   - Before that: Junior Developer at Startup XYZ (2016-2018)
     Python Django web apps, learned Docker
   - Education: BSc Computer Science, University of Manchester, 2016
   - Skills: Go, TypeScript, Python, Kubernetes, Docker, PostgreSQL,
     React, CI/CD, AWS, Terraform
   - Certs: AWS Solutions Architect Associate, CKA
   ```

3. Watch the agent apply formatting skills and produce a polished CV!

### 6.2 — Test with job tailoring

```
Now tailor this CV for this job description:

Title: Platform Engineer at TechCorp
Requirements: 3+ years Kubernetes experience, Go or Rust, infrastructure
as code (Terraform preferred), CI/CD pipeline design, monitoring and
observability, team leadership experience, AWS or GCP
```

## Step 7 — Refine and Iterate

### 7.1 — Add a cover letter skill

```
Create a new skill file: .github/instructions/cover-letter.instructions.md
It should teach the agent how to write a matching cover letter when the
user provides a job description. The cover letter should reference specific
CV achievements that match the role.
```

### 7.2 — Add multiple format output

```
Update the CV Agent so it can output in multiple formats:
- Standard Markdown (default)
- Plain text (for copy-pasting into application forms)
- JSON (structured data for portfolio websites)
The user can specify which format they want.
```

## Summary

| Artifact                                        | Location                  | Purpose                                   |
| ----------------------------------------------- | ------------------------- | ----------------------------------------- |
| `cv-requirements.md`                            | Repository root           | Describes all requirements.               |
| `cv.agent.md`                                   | `.github/agents/`         | The main CV agent.                        |
| `cv-formatting.instructions.md`                 | `.github/instructions/`   | Skill — formatting and ATS rules.         |
| `cv-tailoring.instructions.md`                  | `.github/instructions/`   | Skill — job description tailoring.        |
| `output/cv.md`                                  | Created at runtime        | Final polished CV.                        |

### Key takeaways

1. **Skills are reusable** — the same formatting skill could be used by any document agent.
2. **Instruction files** give agents domain expertise without bloating the agent file.
3. **Practical agents** like CV builders demonstrate immediate personal value.
4. **Iterate quickly** — all files are Markdown, edits take seconds.
